ZapFlood Console

Program for deauth flood

ZapFlood Console is an advanced tool designed to initiate deauthentication (deauth) flood attacks on Wi-Fi networks. These attacks are primarily used to disrupt the communication between wireless devices by continuously sending deauthentication packets to the target network or device. ZapFlood provides an intuitive console interface to execute these attacks with detailed monitoring and control options.

Key Features

  • Customizable Interface Selection: Select the network interface for the attack, supporting various wireless adapters and configurations.
  • Continuous Flooding: Launch a persistent deauthentication flood on the target network or channel until manually stopped.
  • Channel-Specific Attacks: Choose a specific Wi-Fi channel (1-14) for targeted deauth flooding to disrupt the targeted access points more efficiently.
  • Real-Time Monitoring: Track the status of ongoing deauth attacks, including the number of packets sent and targeted devices.
  • Easy-to-Use Console Interface: The console interface is designed to be simple and accessible, with clear instructions and a log output to show the deauth attack status.
  • Network Scanning: Automatically scans for nearby networks and retrieves necessary details, including BSSID, channel, and signal strength (RSSI), which can be stored in JSON files for later use in attacks.
  • Version Information: Check the program’s version using the --version command option for the latest updates.

How It Works

The tool starts by scanning the environment to detect available Wi-Fi networks. During the scan, it switches between channels (1-14) and listens for beacon frames from nearby access points. The detected information, such as the BSSID and channel, is saved into a bssid_list.js file, which is then used to target specific networks during the deauthentication attack.

Once the networks are detected, the program sends a continuous stream of deauthentication packets to the selected BSSIDs on their respective channels. This action forces devices to disconnect, causing temporary disruption of the network. The process can be stopped or modified at any time, depending on the user’s needs.

Advanced Options

  • Live Session Monitoring: View live logs and statistics as the deauth flood is executed, helping to track the progress of the attack.
  • Channel-Specific Attacks: You can target specific channels for a more efficient deauth flood, optimizing the disruption to devices connected to those channels.
  • Multithreading: The tool uses multithreading to increase the efficiency of sending deauthentication packets, ensuring that the attack remains effective even with a high volume of traffic.
  • JSON Logs: All scanning data and attack logs are saved in a JSON format for easy review and analysis, providing detailed information on the networks and devices targeted during the attack.

Usage Instructions

To use the ZapFlood Console, open a terminal window and run the tool with the desired options. The basic command structure is:

zapflood -i/--interface INTERFACE -c/--channel CHANNEL

Additional options include:

  • -i/interface--: Select interface for deauth flood.
  • -s/--still: Run deauth flood continuously.
  • -c/--channel: Set the channel for deauth flood, it will deauth all networks on the selected channel.
  • --live: Console with overview information.
  • --no-color: No Colors.
  • --version: Check the current version of the program.

sudo bash zapfloodInstallationDEBIAN.sh
zapflood --help
